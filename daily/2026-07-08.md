# React 공부하다 JavaScript 정리하기

## 오늘 배운 것

React를 공부하다 보면 React만의 문법이라고 생각하고 넘어가는 경우가 많았다. 하지만 React는 자바스크립트로 작성된 라이브러리이므로, React에서 사용하는 문법은 곧 자바스크립트에서 사용하는 문법과 동일하다.

`export default`는 코드 스니펫을 통해 자동으로 생성되게끔 해서 이게 모듈 시스템의 일부라고만 인지하고 있었는데, 이번에 React 공식 문서를 읽고 정리하는 과정에서 이 문법을 조금 더 깊게 이해하고자 했다.

## 핵심 정리

- 모듈은 하나의 책임만 가지도록 구성해야 한다.
  - 이는 곧 React의 컴포넌트를 설계할 때 적용되는 원칙과 동일하다. (단일 책임 원칙, Single Responsibility Principle)
- `Named Export`와 `Default Export`는 목적이 다르다.
  - `Named Export`는 여러 개의 값을 내보낼 때 사용한다.
  - `Default Export`는 하나의 값만 내보낼 때 사용한다.
- 모듈의 특징
  - 모듈은 독립적인 스코프를 가진다.
  - 모듈은 항상 Strict Mode로 실행된다.
  - 모듈은 한 번만 실행된다.
  - 모듈은 Tree Shaking이 가능해 성능 최적화에 유리하다.

## 다음에 해볼 것

React 공식 문서를 차근차근 읽으면서 이해한 내용과 개념들을 정리할 예정이다. 그리고 중간중간 자바스크립트에서 제공하는 문법이 발견된다면 이를 함께 정리하여 React와 자바스크립트의 관계를 명확히 하고자 한다.
