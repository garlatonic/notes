# 비동기 익숙해지기

## 오늘 배운 것

라이브코딩 스터디 문제를 준비하면서 `Scheduler`와 `curry`를 직접 구현해보는 시간을 가졌다. 스터디에서 구현 과정을 설명할 수 있도록 동작 원리와 구현 의도를 함께 정리한 글의 링크를 남긴다.

[Scheduler 구현하기](../live-coding/class-scheduler.md)
[curry 구현하기](../live-coding/function-curry.md)

## 핵심 정리

### Scheduler

- 실행 중인 작업 수(`running`)와 대기열(`queue`)을 분리하여 관리했다.
- 작업이 성공(`resolve`)하든 실패(`reject`)하든 반드시 다음 작업이 실행될 수 있도록 `finally`에서 실행 슬롯을 반납해야 한다.
- `Promise.resolve().then(task)`를 사용하면 `task()` 내부에서 발생하는 동기 예외까지 하나의 Promise 체인으로 처리할 수 있다.

### Curry

- `Function.prototype.length`를 이용하면 원본 함수가 요구하는 매개변수 개수를 알 수 있다.
- 클로저를 이용해 이전에 전달된 인자를 계속 누적할 수 있다.
- 인자가 충분하지 않으면 다시 함수를 반환하고, 충분해지는 순간 원본 함수를 실행하는 방식으로 구현된다.

## 느낀 점

예전에는 비동기 문제를 풀 때 `Promise`와 `async/await`를 사용하는 것 자체에 집중했다면, 이번에는 **언제 실행되고, 언제 다음 작업이 시작되는지**와 같은 실행 흐름을 더 많이 생각하게 되었다.

특히 `Scheduler`를 구현하면서 `finally`에서 실행 슬롯을 반환하는 이유와 작업 큐가 어떻게 순차적으로 동작하는지를 다시 정리할 수 있었다.
