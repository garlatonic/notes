# 라이브코딩 스터디와 입사 전 학습해야 할 것들

## 오늘 배운 것

라이브코딩 스터디를 진행하면서 어제 열심히 만들었던 `Scheduler` 클래스 문제를 출제했고, 나는 메모이제이션 유틸리티와 URL 쿼리스트링 파서를 구현했다.

또한 입사 전 학습 계획을 정리하는 글도 작성했다. 입사 예정인 회사에서는 글로벌 이커머스 플랫폼 구축에 Shopify를 활용하기 때문에, Shopify의 구조와 개발 흐름을 미리 익혀두는 것이 필요하다고 판단했다.

처음에는 개발자 문서를 처음부터 읽으려고 했지만, 전체 구조를 이해하기 전에 세부 기능부터 학습하는 것은 비효율적이라는 것을 깨달았다. 그래서 Shopify의 구조를 먼저 이해한 뒤 Theme, App, CLI, API 순으로 학습하는 로드맵을 작성했다.

## 느낀 점

오늘 스터디에서 가장 큰 수확은 **다른 사람의 구현 방식을 직접 비교해볼 수 있었다는 점**이다.

메모이제이션과 URL 쿼리스트링 파서를 구현하면서 나는 기능 구현에만 집중했지만, 다른 분들의 코드를 보니 다양한 입력값과 예외 상황까지 고려하여 설계한 부분이 인상적이었다. 구현 자체보다 **어떤 상황까지 고려하며 설계해야 하는지**를 다시 한번 생각해보는 계기가 되었다.

또한 Shopify도 처음에는 막연히 어렵게만 느껴졌는데, 학습 방향을 다시 정리해보니 생각보다 구조가 명확한 플랫폼이라는 것을 알게 되었다. 남은 기간 동안 로드맵에 맞춰 차근차근 학습하여 입사 후에는 프로젝트 구조를 빠르게 파악하고 자연스럽게 적응할 수 있도록 준비해야겠다.
