# 라이브코딩 문제 만들다 공부를 더 해버림

## 오늘 배운 것

라이브코딩 스터디를 대비해 문제를 2개 만들고, 이전 출제된 문제들을 복습하면서 얼렁뚱땅 원리만 파악했던 부분들을 다시 공부했다. prototype, this, 인스턴스 메서드 등... 어떻게 보면 무심코 사용했던 것들이 어떻게 동작하는지에 대해 다시 공부할 수 있는 좋은 기회였다. 특히, `prototype`과 `this`를 집중적으로 공부했다.

## 사이트에 적용한 점

Codex와 Copilot을 이용해 사이트를 **읽기 경험** 중심으로 다듬는 작업을 진행했다.

최근 게시물 노출 수를 늘리고, 검색 결과가 없을 때 안내 문구를 표시했으며, 스켈레톤 로딩을 도입해 문서 로딩 중 공백 느낌을 줄였다. 또 카테고리 레이블을 더 읽기 쉬운 형태로 정리하고, 카드/코드블록/링크 스타일을 손보며 전체 가독성을 개선했다.

### 핵심 정리

- 최근 게시물 영역을 확장해 새 글 탐색이 쉬워졌다.
- 스켈레톤 로딩을 적용해 초기 체감 성능을 개선했다.
- 검색 결과가 없을 때 상태 메시지를 보여줘 사용자 피드백을 보강했다.
- `posts.json` 갱신과 함께 새 라이브코딩 글들을 사이트에 바로 반영했다.

```js
function renderHome() {
  const recent = state.filteredPosts.slice(0, 12);
  elements.recentPosts.innerHTML = recent.length
    ? recent.map(renderPostCard).join("")
    : `<p class="empty-card">검색 결과가 없습니다.</p>`;
}
```
